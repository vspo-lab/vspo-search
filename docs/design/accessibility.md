# アクセシビリティチェックリスト

## 概要

アクセシビリティは、すべてのユーザーがプロダクトを利用できるようにするための重要な要素です。本チェックリストは、WCAG 2.1の基準に基づき、7つのカテゴリで構成されています。

## 1. 代替テキスト

画像やメディアコンテンツに適切な代替テキストを提供します。

| チェック項目 | 詳細 |
|-------------|------|
| 画像に代替テキストが付与されている | すべての `<img>` タグに `alt` 属性を設定する |
| 装飾目的の画像が無視できるようになっている | 装飾画像には `alt=""` または `role="presentation"` を設定する |

### 実装例

```tsx
// 情報を伝える画像
<img src="/article-image.jpg" alt="東京の夜景を上空から撮影した写真" />

// 装飾目的の画像
<img src="/decorative-line.png" alt="" role="presentation" />

// リンク内の画像
<a href="/home">
  <img src="/logo.svg" alt="ホームへ戻る" />
</a>
```

## 2. 動画・音声

動画や音声コンテンツのアクセシビリティを確保します。

| チェック項目 | 詳細 |
|-------------|------|
| 動画の音声に字幕が付与されている | 聴覚障害のあるユーザーが内容を理解できる |
| 動画の内容を解説した音声またはコンテンツがある | 視覚障害のあるユーザーが内容を理解できる |
| 音声や動画、アニメーションが自動で再生される場合、一時停止できる | ユーザーが制御できる |
| 画面内に1秒に3回以上の点滅や閃光を起こすものがない | 光過敏性発作の防止 |

### 実装例

```tsx
<video controls>
  <source src="/video.mp4" type="video/mp4" />
  <track kind="captions" src="/captions-ja.vtt" srclang="ja" label="日本語字幕" />
</video>
```

## 3. マークアップ

セマンティックなHTMLマークアップを使用します。

| チェック項目 | 詳細 |
|-------------|------|
| 表が `<table>` でマークアップされている | レイアウト目的でtableを使用しない |
| 見出しが `<h1>〜<h6>` でマークアップされている | 見出しレベルを順序通りに使用する |
| リストが `<ul>`, `<ol>`, `<dl>` でマークアップされている | 適切なリスト要素を使用する |
| 空白文字を用いてレイアウトをしていない | CSSでレイアウトを制御する |
| 同じ `id` を持つ要素がページ内に複数存在しない | id属性は一意にする |

### 見出しの正しい使用

```tsx
// ✅ OK: 順序通りの見出し
<h1>ページタイトル</h1>
<h2>セクション1</h2>
<h3>サブセクション1-1</h3>
<h2>セクション2</h2>

// ❌ NG: 見出しレベルのスキップ
<h1>ページタイトル</h1>
<h3>セクション1</h3>  {/* h2をスキップ */}
```

## 4. 見やすさ、聞きやすさ、区別しやすさ

視覚的・聴覚的な情報が適切に伝わるようにします。

| チェック項目 | 詳細 |
|-------------|------|
| 画面を200%拡大、または文字サイズを2倍に変更しても情報が取得できる | レスポンシブ対応 |
| 背景色と文字色のコントラスト比が4.5:1以上（大文字29px以上は3:1以上） | 読みやすさの確保 |
| 色や形、音、レイアウト情報のみでコンテンツを説明していない | 複数の手段で情報を伝える |

### コントラスト比の基準

| 対象 | 最小コントラスト比 |
|------|-------------------|
| 通常テキスト（14px未満） | 4.5:1 |
| 大きなテキスト（18px以上 or 14px以上のボールド） | 3:1 |
| UIコンポーネント | 3:1 |

### 色だけに依存しない情報伝達

```tsx
// ❌ NG: 色だけでエラーを示す
<input className="border-red-500" />

// ✅ OK: 色 + アイコン + テキストでエラーを示す
<div>
  <input className="border-red-500" aria-invalid="true" aria-describedby="error-msg" />
  <span id="error-msg" className="text-red-600">
    <ErrorIcon /> 入力内容に誤りがあります
  </span>
</div>
```

## 5. 操作しやすさ

キーボードやその他の入力デバイスで操作できるようにします。

| チェック項目 | 詳細 |
|-------------|------|
| キーボードで操作可能 | すべてのインタラクティブ要素にフォーカスできる |
| キーボード操作の順序が見た目の順序とあっている | tabindexを適切に設定する |
| コンテンツに制限時間がかけられていない | 必要な場合は延長・解除できるようにする |

### キーボードフォーカスの可視化

```css
:focus-visible {
  outline: 2px solid var(--color-border-focus);
  outline-offset: 2px;
}
```

## 6. ナビゲーション

ページ内のナビゲーションを明確にします。

| チェック項目 | 詳細 |
|-------------|------|
| ページの言語が `<html>` に記載されている | `<html lang="ja">` |
| ページのタイトルがページの内容を表している | 具体的で一意なタイトル |
| ページの主要コンテンツに見出しが付与されている | スクリーンリーダーでの移動を容易に |
| リンクのテキストからリンク先が判別できる | 「こちら」のみのリンクは避ける |

### リンクテキストの改善

```tsx
// ❌ NG: リンク先が不明確
<a href="/policy">こちら</a>をご確認ください。

// ✅ OK: リンク先が明確
<a href="/policy">プライバシーポリシー</a>をご確認ください。
```

## 7. フォーム

フォーム入力のアクセシビリティを確保します。

| チェック項目 | 詳細 |
|-------------|------|
| 入力する内容や操作がラベルとして表示されている | `<label>` 要素を使用する |
| フォームパーツにアクセシブルネームがある | `aria-label` または `aria-labelledby` |
| エラーの発生とエラーの内容が特定できる | エラーメッセージを明確に表示する |
| 入力欄や選択肢を選択・入力したときに大きな変化を起こさない | ユーザーの意図しない動作を避ける |

### フォームの実装例

```tsx
<div>
  <label htmlFor="email">メールアドレス</label>
  <input
    id="email"
    type="email"
    aria-required="true"
    aria-invalid={hasError}
    aria-describedby={hasError ? "email-error" : undefined}
  />
  {hasError && (
    <span id="email-error" role="alert" className="text-error">
      有効なメールアドレスを入力してください
    </span>
  )}
</div>
```

## チェックリストの使用方法

1. 開発完了時に本チェックリストを確認する
2. 各項目を検証し、問題があれば修正する
3. 修正後、再度チェックリストを確認する
4. すべての項目がクリアされていることを確認してからリリースする
